# MVP 最小可见版本落地计划（HUD/模型/动画/使用）

> 目标：尽快在游戏内看到“可用 + 可感知”的完整链路，而不是等到所有细节都完美再展示。

## 1. 资产适配什么时候开始？

结论：**现在就该开始（Phase 2 早期并行）**，不要等到网络/渲染全部完成。

推荐双轨并行：

1. **轨道 A：玩法链路**（已在推进）
   - 状态机、会话、输入事件、命中/音画触发
2. **轨道 B：资产适配**（立即启动）
   - 先打通最小资源集：1 套枪模型 + 1 套一人称动画 + 1 套 HUD 图层 + 1 套音效

这样可以避免“代码做完了但看不到效果”的空窗期。

## 2. 最小可见版本（MVP-V0）定义

MVP-V0 以“玩家可完整体验”为准，建议范围：

- **HUD**：弹匣/备弹数字、简版准星、换弹状态提示
- **模型**：一把枪（第一人称 + 基础第三人称）
- **动画**：待机 / 开火 / 换弹（可先简化，不追求最终质量）
- **使用闭环**：
  - 左键开火（含命中反馈）
  - 潜行右键换弹
  - 音效/粒子可见

## 3. 里程碑与进入条件

### M2a（当前 -> 下一步）
- 输入事件 -> 状态机 -> 行为输出链路稳定（已基本具备）
- 加一个 HUD 调试面板显示当前会话状态（gunId、ammo、state）

### M2b（资产最小集接入）
- 接入单枪最小资产包：
  - 模型/贴图路径映射
  - 开火/换弹动画片段绑定
  - HUD 资源绑定
- 通过配置可开关（保留回退）

### M2c（MVP-V0 可玩验收）
- 单人模式：
  - HUD、模型、动画、使用流程完整可见
  - 至少 10 分钟稳定游玩无阻断错误
- 形成“演示构建”给后续网络与渲染深化使用

## 4. 粗略时间预估（按迭代，不是承诺）

- **M2a**：0.5 ~ 1 个迭代
- **M2b**：1 ~ 2 个迭代
- **M2c（首次可见完整最小版）**：通常在 **2 ~ 3 个迭代**

> 注：这里的“迭代”按你当前节奏可理解为 1~2 周一个开发节奏块。

## 5. 风险控制

- 不把所有枪都一次性接入：先单枪样板，模板化后再扩展
- 不追求一次到位动画质量：先保证状态切换正确，再升级表现
- 所有新链路要可回退（feature toggle）
