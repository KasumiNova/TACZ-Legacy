# TACZ UI 体验对齐追踪（Legacy）

> 目标：与 TACZ 原模组 HUD/交互体验保持行为一致；当前按“先闭环可见，再逐项对齐细节”推进。

## 已完成（本阶段）

- [x] 枪械 HUD 基础信息：弹匣/备弹数字、开火模式图标、武器 HUD 图标位
- [x] 准星替换骨架：持枪时替换 Vanilla 准星（可配置）
- [x] 换弹态交互反馈：换弹时隐藏准星并显示中心 `RELOADING`
- [x] 中心进度反馈：TACZ 风格中心条（当前由换弹/冷却驱动）
- [x] 击发反馈：Hit Marker 风格四向标记（当前由射击计数触发）
- [x] 交互提示骨架：瞄准方块时显示换弹提示（可开关）
- [x] 击杀计数骨架：顶部 `KILL xN` 衰减显示（可开关）

## 待对齐（下一批）

- [ ] 过热系统真实数据接入（`heat_max` / `overheat_locked` 行为与颜色闪烁）
- [ ] 命中类型反馈：普通命中 / 击杀 / 爆头颜色与位移逻辑
- [ ] 击杀计数 Overlay（与 TACZ 的持续时间、颜色梯度、触发条件完全一致）
- [ ] 交互提示文案与按键名（`gui.tacz.interact_key.text.desc` 语义与位置对齐）
- [ ] HUD 布局微调（与 TACZ 像素级位置/缩放对齐）
- [ ] RenderConfig 对齐（开关项名称与默认值、跨端同步策略）
- [ ] 资源完整迁移（`textures/hud/*`, `textures/crosshair/*` 实包验证）

## 风险说明

1. 当前 Legacy 尚未接入 TACZ 完整热量数据链路，因此中心条是“语义占位 + 体验先行”。
2. 命中/击杀/爆头需服务端裁决结果回传客户端，避免纯客户端推断导致错报。
3. 为避免回归，后续每项对齐都需配套单测与一轮全量回归。
